## 一、项目概要

公司账号资产管理系统是一个内部工具型 Web 系统，用于统一管理公司持有的各类数字账号资产。  
区别于传统账号管理工具以「账号」为管理单元的设计，本系统以 **「手机设备」为管理核心**，围绕设备→手机号→互联网账号三层递进结构组织数据，实现账号资产的台账化、权限可控、风险可识别。

V1 目标：替代飞书多维表格，完成账号资产台账化并实现敏感信息可控。

### 2.1 用户

| 角色 | 说明 | V1 重点 |
| --- | --- | --- |
| 超级管理员 | 拥有所有权限，含系统设置、用户管理、数据删除 | 是 |

```
  └── ...
```

详细字段定义见第三章。

### 3.2 第一层：设备表（devices）

手机物理设备，作为账号资产管理的入口和主键。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键，自增 |
| device_code | varchar | 系统自动生成编号，格式：DEV-YYYYMMDD-XXXXX |
| device_name | varchar | 设备名称，如"王伟工作机"、"直播专用机" |
| brand_model | varchar | 品牌型号，如 iPhone 14 Pro、Xiaomi 13 |
| imei | varchar | IMEI 码 |
| sim_type | varchar | 枚举：single（单卡）、dual（双卡） |
| owner_subject | varchar | 所属主体，枚举：公司、法人、员工个人 |
| department_id | bigint | 所属部门，FK → departments.id |
| owner_user_id | bigint | 设备负责人，FK → users.id |
| current_user_id | bigint | 当前使用人，FK → users.id，可为空 |
| status | varchar | 枚举：正常、使用中、闲置、异常、已注销 |
| risk_level | varchar | 枚举：低、中、高，自动或手动计算 |
| remark | text | 备注 |
| created_by | bigint | 创建人，FK → users.id |
| updated_by | bigint | 最后修改人，FK → users.id |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted_at | datetime | 软删除时间（V1 始终为 NULL） |

**风险规则（设备级别）**：

- 无负责人（owner_user_id 为空）→ 高风险
- 状态异常（status 为"异常"）→ 高风险
- 状态为使用中但 current_user_id 为空 → 高风险
- 无绑定的手机号 → 中风险

### 3.3 第二层：手机号表（phone_numbers）

每个手机号绑定到设备的某个卡槽，一个手机号可绑定多个互联网账号。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键，自增 |
| device_id | bigint | 所属设备 ID，FK → devices.id |
| slot_type | varchar | 枚举：sim1、sim2（单卡设备固定为 sim1） |
| phone_number | varchar | 手机号，默认脱敏展示 |
| carrier | varchar | 运营商：中国移动、中国联通、中国电信、虚拟运营商 |
| is_primary | boolean | 是否主用号码 |
| monthly_fee | decimal(10,2) | 月套餐费用，可为空 |
| plan_type | varchar | 套餐类型 |
| owner_user_id | bigint | 手机号负责人，FK → users.id |
| status | varchar | 枚举：正常、使用中、闲置、冻结、已注销 |
| remark | text | 备注 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**约束**：

- 单卡设备只能有一条 phone_number 记录（slot_type = sim1）
- 双卡设备最多两条 phone_number 记录（sim1 + sim2）
- 不同设备的卡槽可绑定同一手机号（通过 slot_type 区分）

**风险规则（手机号级别）**：

- 无负责人 → 高风险
- 状态为使用中但 device 的 current_user_id 为空 → 中风险
- 月套餐费用 > 500 元且未标注用途 → 中风险
- 无绑定互联网账号 → 低风险提醒

### 3.4 第三层：互联网账号表（internet_accounts）

手机号下绑定的各类互联网平台账号。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键，自增 |
| phone_number_id | bigint | 绑定手机号 ID，FK → phone_numbers.id |
| account_code | varchar | 系统自动生成编号，格式：ACC-YYYYMMDD-XXXXX |
| platform | varchar | 平台类型，枚举：微信、抖音、小红书、快手、TikTok、Google、QQ、微博、B站、淘宝、京东、拼多多、美团、其他 |
| account_name | varchar | 账号名称（如微信号、抖音昵称） |
| login_account | varchar | 登录账号（需与 phone_number 区分） |
| bind_phone | varchar | 绑定手机号文本（冗余字段，默认从 phone_number_id 派生，可手动覆盖） |
| bind_email | varchar | 绑定邮箱 |
| owner_subject | varchar | 所属主体，枚举：公司、法人、员工个人、客户、代理、其他 |
| purpose | varchar | 用途说明 |
| service_provider | varchar | 服务商名称 |
| monthly_fee | decimal(10,2) | 月费用 |
| expire_at | datetime | 到期时间 |
| department_id | bigint | 所属部门，FK → departments.id |
| owner_user_id | bigint | 账号负责人，FK → users.id |
| current_user_id | bigint | 当前使用人，FK → users.id |
| permission_status | varchar | 枚举：正常、离职待收回、已收回 |
| status | varchar | 枚举：正常、使用中、闲置、冻结、异常、已注销 |
| risk_level | varchar | 枚举：低、中、高 |
| remark | text | 备注 |
| created_by | bigint | 创建人，FK → users.id |
| updated_by | bigint | 最后修改人，FK → users.id |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**风险规则（账号级别）**：

- 无负责人 → 高风险
- 实名信息为空 → 中风险
- 绑定手机号为空 → 中风险
- 状态异常 → 高风险
- 状态为使用中但 current_user_id 为空 → 高风险
- 权限状态为离职待收回 → 高风险
- 人工标记高风险 → 高风险
- 密码超过 90 天未修改 → 中风险（V1 标记，V3 启用规则）

### 3.5 敏感信息表（account_sensitive_infos）

与互联网账号一一对应，敏感字段独立分表且加密存储。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| account_id | bigint | 互联网账号 ID，FK → internet_accounts.id，UNIQUE |
| login_password_encrypted | text | 加密后的登录密码 |
| real_name_info_encrypted | text | 加密后的实名信息（姓名 + 身份证号等） |
| backup_info_encrypted | text | 其他敏感补充信息 |
| password_updated_at | datetime | 密码最近修改时间 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 3.6 操作日志表（operation_logs）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| operator_id | bigint | 操作人，FK → users.id |
| action_type | varchar | 操作类型枚举 |
| target_type | varchar | 目标实体类型：devices、phone_numbers、internet_accounts |
| target_id | bigint | 目标实体 ID |
| before_data | json | 修改前数据（不含敏感字段明文） |
| after_data | json | 修改后数据（不含敏感字段明文） |
| ip_address | varchar | 操作 IP |
| device_info | varchar | 设备/浏览器信息 |
| remark | text | 备注 |
| created_at | datetime | 创建时间 |

**必须记录的操作类型**：

- 新增/编辑/删除（软删）各层级实体
- 修改负责人／当前使用人
- 修改设备 SIM 类型
- 手机号绑定/解绑
- 修改账号状态／风险等级
- 查看敏感信息
- 复制敏感信息
- 导入/导出数据

### 3.7 通用风险记录表（entity_risks）

| detected_at | datetime | 检测时间 |

## 五、关键交互与状态规则

### 5.1 实体层级导航

1. 默认进入「设备列表视图」，展示所有设备
2. 点击设备行 → 右侧详情面板展示设备信息，同时可选「展开」查看该设备下的手机号列表
3. 点击手机号 → 右侧详情面板切换为手机号信息，同时可选「展开」查看该手机号下的互联网账号列表
4. 点击互联网账号 → 右侧详情面板切换为账号信息
5. 在手机号视图和互联网账号视图中，点击行同样驱动右侧详情面板

**导航面包屑**（显示在详情面板顶部）：`设备名称 > 手机号 > 账号名称`，可点击向上层级跳转

### 5.2 搜索

全局搜索框覆盖范围：

| 层 | 可搜索字段 |
| --- | --- |
| 设备 | device_name、device_code、brand_model、imei |
| 手机号 | phone_number（脱敏搜索）、carrier |
| 互联网账号 | account_code、account_name、login_account、platform |

搜索策略：默认模糊搜索，自动匹配所有层级，结果优先显示匹配的设备。

### 5.3 筛选

| 条件 | 适用视图 |
| --- | --- |
| SIM 类型 | 设备视图 |
| 平台类型 | 互联网账号视图 |
| 所属部门 | 所有视图 |
| 负责人 | 所有视图 |
| 状态 | 所有视图 |
| 风险等级 | 所有视图 |

### 5.4 敏感信息脱敏

1. **默认展示**：敏感字段显示为 `••••••`
2. **查看**：点击「查看」按钮 → 权限校验 → 有权限则显示明文 → 写入操作日志
3. **复制**：点击「复制」按钮 → 权限校验 → 有权限则复制到剪贴板 → 写入操作日志
4. **自动恢复**：查看后 30 秒内无任何敏感操作 → 自动恢复为 `••••••`
5. **敏感字段范围**：登录密码、实名信息、完整手机号、绑定邮箱、运营商后台信息、备注中的敏感内容

### 5.5 批量操作

操作规则：

- 仅当勾选至少一条数据时显示批量操作栏
- 批量操作只适用于「互联网账号视图」
- 批量收回权限：将选中账号的 permission_status 改为"已收回"
- 批量移交负责人：弹窗选择新负责人 → 批量修改
- 批量标记高风险：将选中账号的 risk_level 改为"高"
- 导出所选：导出选中行数据到 CSV/Excel

### 5.6 导入导出

**导入**：

- 支持 CSV 和 Excel 格式
- 支持多层导入：一次导入可包含设备 + 手机号 + 互联网账号
- 导入模板按层分 sheet：Sheet1=设备、Sheet2=手机号（参考设备编号）、Sheet3=互联网账号（参考手机号 ID 或手机号原文）
- 导入前预览展示映射关系
- 导入后展示成功/失败行数及错误原因
- 包含导入时触发风险扫描
- 批量导入的创建人为当前操作人

**导出**：

- 账号管理员、超级管理员可导出全量
- 部门负责人仅导出本部门
- 普通员工不可导出
- 导出行为写入操作日志
- 支持导出当前视图数据或所选行

## 六、设计注意事项

### 6.1 数据一致性

- 删除设备时需级联处理：检查下属手机号和互联网账号，提示用户先解绑或转移
- 修改设备 SIM 类型（single→dual 或 dual→single）时需校验：降级为单卡时需确认 sim2 手机号的处理逻辑
- 设备负责人变更时，下属手机号和互联网账号的负责人可选择同步更新或保持原值

### 6.2 权限映射

原 PRD 的 5 种角色权限体系保持不变，但数据可见范围需映射到三层：

| 角色 | 设备 | 手机号 | 互联网账号 |
| --- | --- | --- | --- |
| 超级管理员 | 全量 | 全量 | 全量 |
| 老板 | 全量（只读） | 全量（只读） | 全量（只读） |
| 账号管理员 | 全量 | 全量 | 全量 |
| 部门负责人 | 本部门设备 | 本部门相关 | 本部门账号 |
| 普通员工 | 与自己相关的 | 与自己相关的 | 授权/使用/申请过的 |

### 6.3 API 设计变更（与原始 PRD 对比）

原始 PRD 的 API 基于 accounts 表设计，需重构为三层 API：

| 层级 | 主要端点（建议） | 说明 |
| --- | --- | --- |
| 设备 | /api/devices | CRUD + 列表 + 搜索 |
| 手机号 | /api/phone-numbers | CRUD，参数含 device_id |
| 互联网账号 | /api/internet-accounts | CRUD + 批量操作 |
| 敏感信息 | /api/internet-accounts/{id}/sensitive | 查看/复制/修改 |
| 操作日志 | /api/operation-logs | 查询、导出、按实体过滤 |
| 风险 | /api/risks | 全量风险 + 按实体过滤 + 手动扫描 |
| 导入 | /api/import | 支持多层导入 |
| 导出 | /api/export | 按层级 + 权限导出 |

### 6.4 UI/UX 注意点

- 三栏布局固定宽度，不做响应式折叠。如果屏幕过窄，中间主区出现横向滚动条而非折叠侧栏
- 三种视图（设备/手机号/账号）通过 Tab 切换，当前视图高亮，自动刷新列表
- 右侧详情面板内容跟随选中实体动态变化，面板顶部展示面包屑导航和「关闭面板」按钮
- 新增设备/手机号/账号统一使用居中弹窗形式
- 手机号和互联网账号的新增功能，既可在设备详情页内操作，也可在手机号/账号视图中通过独立的新增按钮操作

### 6.5 事务与级联

- 新增设备后，可选「同时绑定手机号」：在设备创建成功后弹出手机号绑定引导弹窗
- 新增手机号后，可选「同时添加互联网账号」：引导用户继续操作
- 批量导入如分多层 sheet，需在事务中按「设备→手机号→互联网账号」顺序导入，任一环节失败则整批回滚

### 6.6 UI 组件复用规划

以下组件可在多个视图/弹窗中复用：

- SensitiveField 敏感字段组件（脱敏/查看/复制/30s 恢复/日志）
- EntityDetailPanel 右侧详情面板容器（根据 entity_type 动态加载不同区块）
- EntityTable 表格组件（支持选择、排序、筛选、分页）
- EntityFormModal 编辑弹窗（根据实体类型动态生成表单）
- RiskBadge 风险等级标签
- StatusBadge 状态标签
- BreadCrumb 面包屑导航组件
